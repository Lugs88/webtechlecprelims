HTTP

HTTP

HTTP is based on a client-server architecture.

  • Servers - origin server
  • Clients also know as (User Agents) - web browsers, web crawlers

In addition, a communication protocol used to access resources on the World Wide Web. Invented by Tim-Berners Lee at CERN in 1989.

HTTP with its Version History:

  • HTTP 0.9(1991)
  • HTTP 1.0(RFC 1945, May 1996) - get, header, post
  • HTTP 1.1(RFC 2068 Jan 1997; RFC 2616 June 1999), RFC 7230 - 7235(June 2014)
  • HTTP 2(RFC 7540 May 2015) - additional mechanism in the transmission of data with header compression that allows multi pleasing which is more enhanced than the previous versions.

HTTP uses a request message. In addition, it is a stateless communications protocol. We have here its functionalities:

  • cache control
  • content media type(MIME)
  • persistent connection - it can be reuse
  • request pipelining/multiplexing - can have many request at a time.
  • authentication/authorization - talks about the security and identifies if the client can be verified and authorized at the same time.

STANDARD METHOD

  • GET - fetch or simply select requiremen to all servers.
  • HEAD - retreive only metadata possibly to validate data
  • POST - perform resource, specific processing. In addition, if there is a post, there is a message body and is mostly used in submitting HTML from data.
  • PUT - hard to implement
  • DELETE - hard to implement
  • OPTIONS - tells what you can do
  • TRACE - allows the server to re-echo the request
  • CONNECT - create a secure connection and establish a tunnel.

SAFE METHOD

Like read only, no expected change just simply fetching

  • GET, HEAD, OPTIONS, TRACE

Idempotent Methods

Intended effect on the server of multiple identical request.

  • GET, HEAD, OPTIONS, TRACE, PUT, DELETE

Cacheable Methods

Indicates that the response to a method is allowed to be stored for future use. In addition, it also saves on a bandwidth.

  • GET, HEAD, POST

STATUS CODE

Informational(1xx)

Message Description
Continue(100) Gets a response that its OK and the client can proceed with the request or ignore if its already done.
Switching Protocol(101) Asks the server if it can switch protocol.

Success(2xx)

Ok(200) The request succeeded.
Created(201) The request has been fulfilled and a new source is created.
Accepted(202) The request has been accepted but its not yet done.
No Content(204) Successfully processed but doesn't return any content.

Redirection(3xx)

Multiple Choices(300) The request has many possible response.
Moved Permanently(301) The requested page has moved to a new URL.
Found(302) The requested page has moved to a new URL temporarily.
See Other(304) The requested page can be found in a different URL
Not Modified(305) The requested page has not been modified.

Client Error(4xx)

Bad Request(400) The request cannot be understand due to invalid syntax.
Unauthorized(401) The requested is available but the server refuse to respond.
Forbidden(403) The client doesn't have access to the right content.
Not Found(404) The server can't find the requested resource.
Request Timeout(408) The server is time out and is waiting for the request.

Server Error(5xx)

Internal Server Error(500) Error message, which the server doesn't how to handle.
Bad Gateway(502) Received an invalid response from the hanlder request.
Service Unavailable(503) The server is not available.
Gateway Timeout(504) The error response is given when the server is acting a gateway and can't get a response in time.
